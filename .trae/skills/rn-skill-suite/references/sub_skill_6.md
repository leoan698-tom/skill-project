# 目的
沉淀优秀组件的标准化文档与类型，提升后续复用与生成质量。

# 何时使用
- 新组件质量达标并需要纳入知识库时。

# 输入
- 组件名称、路径、特征梳理、类型与示例需求。

# 步骤
1. 提取核心特征（功能、关键 Props、样式要点、场景）。
2. 生成 .d.ts 类型定义文件内容。
3. 生成 3~5 个典型使用示例代码。
4. 列出所有变体及其区别（如适用）。

# 提示词模板
```text
刚才生成的 [组件名] 质量很高,符合我们的设计规范。

请帮我做以下整理:

1. 提取该组件的**核心特征描述**(100 字以内)
   - 主要功能
   - 关键 Props
   - 样式特点(高度/圆角/间距等)
   - 使用场景

2. 生成该组件的**类型定义文件** (.d.ts)

3. 生成该组件的**使用示例代码**(3~5 个典型场景)

4. 如果该组件有变体,列出所有变体及其区别

格式示例:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▸ components/product/ProductCard.tsx
作用: 商品卡片,用于列表/推荐位展示
关键特征:
- 左图右文布局,图片 80x80
- 标题最多 2 行,title-md 字体
- 价格 danger 色强调,display-md
- 支持可选的原价划线 + 促销标签
- 整体可点击,有触碸反馈
Props:
- imageUrl: string
- title: string
- price: number
- originalPrice?: number
- badge?: string
- onPress: () => void
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请按此格式输出,我将把它加入组件知识库。
```

# 输出要求
- 标准化的组件特征摘要、类型定义与示例代码。
- 清晰列出变体及差异（如适用）。

# 文档与协作要求
- 类型与文档：为每个 Hook/复杂组件添加 JSDoc；Props 接口添加说明注释；输出 .d.ts 类型定义
- 使用示例：提供 3~5 个典型场景示例，覆盖常见变体与交互
- 结构与命名：遵循模块化目录与命名约定（组件 PascalCase，Hook 以 use 前缀的 camelCase，styles.ts/types.ts 固定命名）
- 条件渲染安全性：示例中统一使用 `!!` 或三元；数字显式比较；字符串长度检查
- Toast 规范：示例中禁止使用 Modal 实现 Toast/Snackbar；采用绝对定位 + zIndex/elevation
